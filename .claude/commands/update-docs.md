다음 순서로 문서를 검토하고 업데이트하세요.

1. `docs/PRD.md` 읽기 — 기능 요구사항이 현재 구현과 일치하는지 확인
2. `docs/TECHNICAL_DESIGN.md` 읽기 — 기술 스택, 폴더 구조, 상태 관리 설계가 실제 코드와 일치하는지 확인
3. `README.md` 읽기 — 실행 방법, 스택 목록, 설계 결정이 최신 상태인지 확인
4. 현재 코드 구조(`src/`, `e2e/`, `src/mocks/`) 파악
5. 불일치하거나 빠진 내용이 있으면 해당 문서에 직접 반영
6. 변경한 항목 목록을 간략히 보고

문서 수정 기준:
- 구현되지 않은 내용을 "구현 예정" 또는 삭제
- 새로 추가된 패턴이나 결정 사항 반영
- 테이블, 목록 등 마크다운 포맷 일관성 유지
- 불필요한 내용 추가 금지, 현재 상태 정확히 반영
